Meta and embeds for Discord (and other social media sites)

Introduction

From services (Servers Lists, as an example) to server/bot websites, there are many websites related to discord. As an example, take a look at the Namecheap listing for "Discord" or at the list of domains backlinking to "discordapp.com" - Crazy right?

The interesting part is that since these websites are related to Discord they are mostly shared on the eponymous platform. Sounds logic to you? Excellent, because that's what this post is going to focus on.

Indeed, we are going to take a look today at how to optimize the advertising power of your website on Discord through the magic of embeds.

Before going any further, I assume most of you are aware of what an embed is. However, it is important to acknowledge that we are not talking about rich embeds created by a bot or a webhook, but about this kind of embeds. Got it? Wonderful, let's begin then!

Let's begin!

In order to facilitate the reading of this post and make it accessible to as many people as possible, I will consider that (while being slightly familiar with HTML and somehow resourceful) you have never bothered to add an embed on your website.

Requirements

  • Access to an online hosted .html file

Remember the embed that I showed you earlier? That's what Discord displays, using only OpenGraph meta tags. You can take a look at the list of meta tags on their website, but actually Discord doesn't support most of them. Actually, the only meta tags supported by Discord with "OpenGraph" are the following:

og:title, og:site_name, og:image, and og:description

These are also all the tags used by Discord on the said embed. Now, let's try to reproduce it ourselves, shall we? To begin with, simply create an html file and fill it with the proper syntax for meta tags as shown below. Then, add your own text. I personally reused discordapp.com's embed, in order to showcase how it's created.

<meta content="Discord - Free Voice and Text Chat" property="og:title"> <meta content="Step up your game with a modern voice & text chat app. Crystal clear voice, multiple server and channel support, mobile apps, and more." property="og:description"> <meta content="Discord" property="og:site_name"> <meta content='https://discordapp.com/assets/ba74954dde74ff40a32ff58069e78c36.png' property='og:image'>

Alright, let's take a look now. Wonderful! These embeds, are looking like exactly the same.. Well, except that I took the wrong asset for the image but.... Shhh....

Did you know?- As the content above is HTML, it does not matter whether you use this syntax:

<meta content="Discord" property="og:site_name">

Or this syntax:

<meta property="og:site_name" content="Discord">

We need to go deeper!

That's dope, right? We got a nice looking little embed for our website. But... it's too generic. Let's see what else we can do, alright?

Requirements

  • Access to an online hosted .html file.

  • Access to an online hosted .json file.

Previously, we used "OpenGraph" to display our embed. But, as you noticed it's not that exciting. Not at all, in fact. Which is why we are now going to add oEmbed on top of that! Thanks to oEmbed, we will be able to add a new field, as well as editing a little bit further an already existing field. In an other category, we will use more in depth oEmbed functions to improve our embed even further.

To begin with, let's see how to call oEmbed tags inside our html page. Unlike the previous ""built-in"" meta tags, this one will require us to use the tag. That is why we will need an additional json file, on top of the already existing html file. But first, let's take a look at (some of) the tags provided by oEmbed:

title, author_name, author_url, provider_name, provider_url.

That is not the full list of tags, like OpenGraph, but rather some of the tags supported by Discord. You can browse the complete list on their website. We will however, talk about some tags such as type a little bit further. First, let's try to create an embed without mixing both oEmbed and OpenGraph, to see what is provided by oEmbed.

{
"title": "Title Field",

"author_name": "Author Name Field",
"author_url": "https://discordapp.com",

"provider_name": "Provider Name Field",
"provider_url": "https://discordapp.com"
}

Once we created our json file, let's say oembed.json we will need to host it. If you are using a webserver, simply call it with /oembed.json but if you are hosting it on a gist, you will need to include the full url.

<link type="application/json+oembed" href="where-the-oembed-info-is.json">

In my case, I hosted the JSON file on an external ressource, and I called it in my html page like that:

<link type="application/json+oembed" href="https://owo.whats-th.is/26822d.json">

And this is how it looks like. Wonderful, right? We earned an additional field, as promised. The "Author Name" field. And, if we click on each one of them, you will notice that the "Provider Name" field is now clickable, unlike with the openGraph meta tags. However, and this time like the openGraph meta tags, we are not able to set a custom url for the Title field. No big deals, right? You need to link to your website anyways.

Now, here comes the fun. As you noticed, the openGraph meta tags can add images and description, while the oEmbed tags can add an Author field and a clickable provider field. So... let's mix both of them! First of all, let's just take a quick look at the following sheet to know when to use openGraph, and when to use oEmbed:

Description Image Author Field Provider Field Title
openGraph ✔️ ✔️ ❌ ✔️
oEmbed ❌ ❌ ✔️ - Clickable ✔️ - Clickable

Alright, so now let's build an enhanced embed. Sounds good to you? I am going to take the ""same"" example embed as before, with a few added features.

JSON

{
    "author_name": "It's time to ditch Skype and TeamSpeak.",
    "author_url": "https://discordapp.com",
    "provider_name": "We are hiring! Click here to join us.",
    "provider_url: "https://discordapp.com/jobs"
}

HTML

<meta content="Discord = Free Voice and Text Chat" property="og:title">
<meta content="All-in-one voice and text chat for gamers that's free, secure, and works on both your desktop and phone. Stop paying for TeamSpeak servers and hassling with Skype. Simplify your live." property="og:description">
<meta content="https://discordapp.com/assets/ba74954dde74ff40a32ff58069e78c36.png" property="og:image">
<link type="application/json+oembed" href="https://owo.whats=th.is//e61180.json">

Result

And here's how it looks like. I agree, it's not beautiful because there is too much information. I just wanted to showcase different possible usages of the field. Obviously, you do not have to keep all the fields. You could as an example, go without the provider field, as shown here.

By the way, did you notice something? Yup, I added a newline in the description field. It's pretty cool actually, because you can make much cleaner looking fields. I'll let you play with that.

I said deeper! That's not deep enough!

Ooooh... I see you're a curious man. Because, yes, we can indeed go muuuch deeper. Yaaay!

Colouring the embed sidebar.

Yup. That's right. You, can color the sidebar of your embed. Sounds too good to be true? Try it by yourself, then!

This trick doesn't require oEmbed or openGraph, since it's a default meta tag. To define a custom sidebar color, you'll need an #HEX code. I personally went with discordapp's blurple. To the HTML codeblock above, I added the following line:

<meta name="theme-color" content="#7289DA">

Let's upload the file and... woah! - We got a fresh coat of paint on that dirty embed. Wonderful, right?

Moving the current image to the bottom

I agree with you, the current embed doesn't looks fantastic with that super small banner. So.. let's fix that!

This trick requires the oEmbed "type" tag I talked about before as well as a few others. Warning! - Doing that will remove the description field. To do that, add the following line to your json file:

<meta name="twitter:card" content="summary_large_image">

It will result in the following embed. Much better, right?
